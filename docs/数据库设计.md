# 数据库设计（MySQL + MongoDB）

## 1. 关系型数据库（MySQL）

## 1.1 users（用户表）
| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| user_id | INT | PK, AUTO_INCREMENT | 用户主键 |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名 |
| password | VARCHAR(255) | NOT NULL | BCrypt 哈希密码 |
| name | VARCHAR(50) | NULL | 姓名 |
| phone | VARCHAR(20) | NULL | 电话 |
| education | VARCHAR(100) | NULL | 学历 |
| skills | TEXT | NULL | 技能标签（逗号分隔） |
| experience | TEXT | NULL | 工作经历 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |

## 1.2 jobs（职位表）
| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| job_id | INT | PK, AUTO_INCREMENT | 职位主键 |
| title | VARCHAR(100) | NOT NULL | 职位名称 |
| company | VARCHAR(100) | NOT NULL | 公司 |
| salary | VARCHAR(50) | NULL | 薪资范围 |
| education | VARCHAR(50) | NULL | 学历要求 |
| experience | VARCHAR(50) | NULL | 经验要求 |
| skills_required | TEXT | NULL | 技能要求 |
| description | TEXT | NULL | 职位描述（可拆分到 MongoDB） |
| location | VARCHAR(100) | NULL | 工作地点 |
| industry | VARCHAR(50) | NULL | 行业 |
| publish_time | DATETIME | NULL | 发布时间 |
| job_url | VARCHAR(255) | NULL | 原始链接 |

## 1.3 skill_dict（技能词典）
| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| skill_id | INT | PK, AUTO_INCREMENT | 技能主键 |
| skill_name | VARCHAR(50) | UNIQUE, NOT NULL | 技能名称 |
| category | VARCHAR(50) | NULL | 分类 |
| weight | FLOAT | NULL | 权重 |

## 1.4 user_actions（用户行为）
| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | BIGINT | PK, AUTO_INCREMENT | 记录主键 |
| user_id | INT | FK, NOT NULL | 用户ID |
| job_id | INT | FK, NOT NULL | 职位ID |
| action_type | VARCHAR(20) | NOT NULL | click/favorite/view |
| action_time | DATETIME | NOT NULL | 行为时间 |

## 1.5 favorites（收藏记录）
| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | BIGINT | PK, AUTO_INCREMENT | 记录主键 |
| user_id | INT | FK, NOT NULL | 用户ID |
| job_id | INT | FK, NOT NULL | 职位ID |
| favorite_time | DATETIME | NOT NULL | 收藏时间 |

## 1.6 recommendations（推荐记录）
| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | BIGINT | PK, AUTO_INCREMENT | 记录主键 |
| user_id | INT | FK, NOT NULL | 用户ID |
| job_id | INT | FK, NOT NULL | 职位ID |
| score | FLOAT | NULL | 推荐分数 |
| recommend_time | DATETIME | NOT NULL | 推荐时间 |

## 2. 非关系型数据库（MongoDB）

## 2.1 job_texts（建议集合）
- 用途：存储职位长文本、原始抓取片段、解析元信息。
- 建议字段：
  - `job_id`（与 MySQL 对齐）
  - `raw_html`
  - `desc_clean`
  - `extra_tags`
  - `updated_at`

## 3. 索引建议
- `users.username` 唯一索引。
- `jobs.title`、`jobs.company`、`jobs.publish_time` 普通索引。
- `user_actions(user_id, action_time)` 组合索引。
- `favorites(user_id, favorite_time)` 组合索引。
- `recommendations(user_id, recommend_time)` 组合索引。

## 4. 初始化策略
- 使用迁移脚本创建结构。
- 使用 seed 脚本导入最小演示数据（建议 500~2000 条职位）。
- 所有时间字段统一使用北京时间或 UTC+8 存储策略。
