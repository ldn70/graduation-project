# 两天重建执行计划

## 1. 目标与约束
- 目标：在 2 天内完成“可演示、可答辩”的前程无忧招聘数据分析系统最小可用版本（MVP）。
- 约束：时间极短，优先保证主流程可跑通，再做算法增强和性能优化。
- 成功标准：
  - 用户可注册/登录。
  - 可搜索职位并查看推荐结果。
  - 可查看技能分析、薪资预测、趋势分析图。
  - 可一键生成并下载简历。
  - 核心接口可联调，具备基础安全与异常处理。

## 2. 范围界定（两天内）
- 必做：
  - 后端 API 主干、MySQL 核心表、JWT 鉴权。
  - 招聘搜索、推荐、技能分析、薪资预测、趋势分析、简历生成。
  - Vue 前端基础页面与 ECharts 可视化。
  - Docker Compose 本地一键启动。
- 延后：
  - 大规模分布式爬虫集群真实部署。
  - Spark/HDFS 全链路生产级调度。
  - 复杂深度学习推荐模型替换。

## 3. 日程分解

## Day 1 上午（4 小时）
- 初始化项目结构：
  - `backend/` Django + DRF。
  - `frontend/` Vue3 + Axios + ECharts。
  - `docker-compose.yml`（mysql、redis、backend、frontend）。
- 建立基础能力：
  - 环境变量与配置分层（dev/prod）。
  - 通用响应格式、中间件、全局异常处理。
  - JWT 认证与权限基类。

## Day 1 下午（4 小时）
- 完成数据库与用户体系：
  - 建表：`users/jobs/skill_dict/user_actions/favorites/recommendations`。
  - 用户接口：注册、登录、更新资料、注销。
- 完成招聘搜索：
  - 关键词模糊匹配。
  - 学历/薪资/分页筛选。
- 前端联调：
  - 登录注册页。
  - 搜索页列表展示。

## Day 1 晚上（2-3 小时）
- 完成简历模块：
  - 简历生成接口与下载接口。
  - 前端“个人中心-生成简历”联调。
- 写入初始职位样例数据（seed），保证 Day 2 算法可调试。

## Day 2 上午（4 小时）
- 完成推荐与技能分析：
  - CBF（TF-IDF + 余弦）基线。
  - CF 简化逻辑（行为相似用户候选）。
  - Hybrid 融合（`alpha=0.6`）。
  - 技能需求分析、技能匹配度接口。
- 前端联调：
  - 推荐页。
  - 技能分析页（柱状图/雷达图）。

## Day 2 下午（4 小时）
- 完成预测模块：
  - 薪资预测接口（XGBoost 预训练模型 + 区间输出）。
  - 趋势分析接口（Prophet 基线，LSTM 预留扩展位）。
- 前端联调：
  - 薪资预测页。
  - 趋势分析页（历史+预测折线）。

## Day 2 晚上（2-3 小时）
- 集成测试与压测：
  - 功能回归（按测试用例表执行）。
  - 安全测试（Token、注入、CORS）。
  - 性能冒烟（搜索/登录/推荐）。
- 交付材料：
  - README 运行文档。
  - API 文档与答辩演示路径。

## 4. 角色与并行建议
- 后端主线：数据模型、接口、算法服务。
- 前端主线：页面与图表组件、接口联调。
- 数据主线：样例数据、模型文件、脚本。
- 若单人开发：严格按“先后端接口可用，再前端联调”的顺序推进。

## 5. 风险与应急预案
- 风险：模型训练耗时超预期。
  - 预案：使用离线预训练模型文件，先保证推理接口可用。
- 风险：爬虫数据不足或不可采。
  - 预案：使用样例数据与历史 CSV 快速导入。
- 风险：前后端联调卡住。
  - 预案：先用 Postman 完成后端验收，再替换前端 mock。
- 风险：性能不达标。
  - 预案：启用分页、字段裁剪、索引与 Redis 缓存。
