# 统一错误码规范

更新时间：2026-04-16

## 1. 目标与范围
- 本规范用于统一后端错误码设计，并为前端提供稳定、可回归的提示映射依据。
- 适用范围：`users/jobs/analysis/resumes` 模块的 API 错误响应。
- 本文档覆盖三部分：
  - 错误码命名规范
  - 接口错误码覆盖清单（当前实现）
  - 前端映射表与兜底规则

## 2. 统一响应约定
- 成功响应（示意）：
```json
{
  "success": true,
  "message": "ok",
  "data": {},
  "statusCode": 200
}
```
- 失败响应（示意）：
```json
{
  "success": false,
  "message": "参数错误",
  "data": {},
  "statusCode": 400,
  "code": "JOB_SEARCH_PAGE_INVALID"
}
```
- 说明：
  - `code` 为可选字段，仅在错误语义明确且需要前端稳定映射时返回。
  - 后端统一响应工具位于 `backend/core/response.py`（`error_response(..., code=...)`）。

## 3. 命名规范

### 3.1 命名格式
- 统一格式：`<MODULE>_<FIELD_OR_SCENE>_<ERROR_TYPE>`
- 字符规则：全大写、下划线分隔、仅使用英文字符和数字。

### 3.2 模块前缀（建议）
- `USER`：用户域（注册/登录/资料）
- `JOB_SEARCH`：职位搜索域
- `RECOMMEND`：推荐域
- `SKILL_DEMAND`、`SKILL_MATCH`：技能分析域
- `RESUME`：简历域
- `COMMON`：全局异常域（鉴权、权限、路由不存在、服务异常等）

### 3.3 错误类型后缀（建议）
| 后缀 | 语义 | 建议 HTTP 状态 |
| --- | --- | --- |
| `INVALID` / `INVALID_PARAMS` | 参数格式或取值非法 | 400 |
| `REQUIRED` / `MISSING` | 缺少必填参数 | 400 |
| `NOT_FOUND` | 目标资源不存在 | 404 |
| `CREDENTIALS_INVALID` | 凭证错误 | 401 |
| `UNSUPPORTED` | 不支持的格式或能力 | 400 |

### 6.2 前端兜底规则
- 若后端未返回 `code`，前端按以下顺序兜底：
  1. 按状态码：`401/403`（权限不足）、`429`（频率限制）、`5xx`（服务异常）
  2. 按超时：`ECONNABORTED` 或 `timeout/超时` 关键字
  3. 按文案关键字推断 `参数缺失/格式错误`
  4. 通用错误提示
- 若后端返回未知 `code`，前端按模式规则兜底：
  - `*_REQUIRED` / `*_MISSING` -> 参数缺失
  - `*_INVALID` / `*_FORMAT` -> 格式错误
  - `*_NOT_FOUND` -> 资源不存在
  - `COMMON_*` 建议优先显式映射，避免被后缀规则误判语义

## 7. 变更流程（新增错误码）
1. 后端新增错误分支时，优先定义稳定 `code`，避免只返回自由文本 `message`。
2. 在 `backend/*/tests.py` 增加对应断言（`resp.data["code"]`）。
3. 在前端 `ERROR_CODE_HINTS` 中新增映射，确保提示可行动。
4. 更新本文档第 4/5/6 节，保持“实现-测试-文档”一致。

## 8. 分级限流策略（当前默认）
- 配置位置：`backend/config/settings.py`（`DEFAULT_THROTTLE_CLASSES` + `DEFAULT_THROTTLE_RATES`）。
- 环境变量示例：`backend/.env.example`。
- 关键 scope 与默认速率：
  - `auth_register`: `10/min`
  - `auth_login`: `20/min`
  - `jobs_search`: `180/min`
  - `analysis_recommend`: `120/min`
  - `analysis_skill_demand`: `150/min`
  - `analysis_skill_match`: `120/min`
  - `analysis_salary_predict`: `90/min`
  - `analysis_trends`: `120/min`

  - `resume_download`: `180/min`

  - 登录/注册接口速率更严格，用于降低暴力尝试风险。
  - 查询类接口速率更宽松，保障正常交互流畅。
